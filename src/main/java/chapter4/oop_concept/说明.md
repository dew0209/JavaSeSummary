oop的基本概念

```xml
面向对象的程序是由对象组成的，每个对象包含用户对公开的特定功能部分和隐藏的实现部分。在oop中，不必关心对象的具体实现，只要能够满足用户的需求即可。
对于一些规模较小的问题，将其分解为过程的开发方式比较理想。而面向对象更加适用解决规模较大的问题。
类是构建对象的模板或蓝图。由类构造对象的过程称为对象的实例化。
封装[数据隐藏]：将数据和行为组合在一个包中，并对对象的使用者隐藏了数据的实现方式。对象中的数据称为实例域，操纵数据的过程称为方法。对于每个特定的类实例（对象）都有一组特定的实例域值。这些值的集合就是这个对象的当前状态。程序仅通过对象的方法与对象数据进行交互[方法调用]。这意味着一个类可以全面地改变存储数据的方式，只要仍旧使用同样的方法操作数据，其他对象就不会知道或所介意所发生的变化。
继承：会让用户自定义Java类变得轻而易举：可以扩展一个类来建立另外一个新的类。这个扩展后的新类具有所扩展的类的全部属性和方法。在新类中，只需要提供适用于这个新类的新方法和数据域就可以了。
三个主要特性[不是原则]:
	对象的行为：可以对对象施加那些操作，或可以对对象施加哪些方法
	对象的状态：当施加那些方法时，对象如何响应。
	对象标识：如何辨别具体相同行为与状态的不同对象
注意：
	1.的状态可能会随着时间而发生改变，但这种改变不会是自发的。对象的状态的改变必须通过调用方法实现(如果不经过方法调用就可以改变对象状态，只能说明封装性遭到了破坏)
	2.作为一个类的实例，每个对象之间的标识永远是不同的，状态常常也存在着差异
	3.对象的这些关键特性彼此之间相互影响着
```

类之间的关系

```xml
依赖(uses-a):如果一个类的方法操纵另一个类的对象，我们就说一个类依赖于另一个类
聚合(has-a)：类A的对象包含类B的对象
继承(is-a)：类A扩展类B，类A不但包含从类B继承的方法，还会拥有一些额外的功能。
```

使用预定义类

```xml
想要使用对象，就必须首先构造对象，并指定其初始状态。然后，对对象应用方法。
在Java中，使用构造器（constructor）构造新实例。
	构造器：用来构造并初始化对象
		构造器的名字应该与类名相同：new Date();
	通常，希望构造的对象可以多次使用，因此，需要将对象存放在一个变量中。Date birthday = new Date();
一定要认识到，一个对象变量并没有实际包含一个对象，而仅仅是引用一个对象
任何对象变量的值都是对存储在另外一个地方的一个对象的引用。new 操作符的返回值也是一个引用。
```
