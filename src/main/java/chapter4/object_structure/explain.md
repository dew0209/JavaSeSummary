重载

```xml
有些类可以有多个构造器。这种特征叫做重载。
编译器通过用各个方法给出的参数类型与特定方法调用所使用的值类型进行匹配来挑选出相应的方法。如果编译器找不到匹配的参数，就会产生编译时错误，因为根本不存在匹配，或者没有一个比其它的更好。--->重载解析
Java允许重载任何方法，而不只是构造器方法。因此，要完整的描述一个方法，需要指出方法名以及参数类型。
```

默认域初始化

```xml
如果没有在构造器中显示地给域赋值，那么就会被自动地赋值：数值为0，布尔值为false，对象引用为null。
注：
	这是域与局部变量的主要不同点。必须明确地初始化方法中的局部变量。
推荐显式的去给域赋值，不要等到对象创建以后调用相应的getXxx方法
```

无参数的构造器

```xml
很多类都包含一个无参数的构造函数，对象由无参函数构造时，其状态会设置为适当的默认值。例如：
	public Employee{
		name = "";
		salary = 0;
		hireDay = LocalDate.now();
	}
如果在编写一个类时没有编写构造器，那么系统会默认提供一个无参数构造器。这个构造器将所有的实例域都设置为默认值。
如果类中提供了至少一个构造器，但是没有提供无参数构造器，则在构造对象如果没有提供参数就会被视为不合法。
```

显式域初始化

```xml
通过重载类的构造器方法，可以采用多种形式设置类的实例域的初始状态。
可以在定义类中，直接将一个值赋给域。private String name = "xxx";[在构造器之前，先执行赋值操作]
可以调用方法对域进行初始化操作。
	public class Employee {
    private static int nextId;
    private int id = assign();
    //可以是static，也可以不是static修饰
    private static int assign(){
        return nextId++;
    }

    public static int getNextId() {
        return nextId;
    }

    public  int getId() {
        return id;
    }
	}
```

调用另一个构造器

```xml
关键字this引用方法的隐式参数。还有另外一层含义：
如果构造器的第一个语句形如this(...),这个构造器将调用同一个类的另一个构造器。
这样对公共的构造器代码部分只编写一次即可。
```

初始化块

```xml
初始化数据域的一种方式。
由于初始化数据域有多种途径，所以列出构造过程的所有路径可能相当混乱。下面是调用构造器的具体处理步骤
	1.所有数据与被初始化为默认值[0,false,null]
	2.按照类声明中出现的次序，依次执行所有域初始化语句和域初始化块
	3.如果构造器第一行调用了第二个构造器，则执行第二个构造器
	4.执行这个构造器的主体。
```

对象析构与finalize方法

```xml
可以为任何一个类添加一个finalize方法。finalize方法将在垃圾回收器清楚对象之前调用。在实际应用中，不要依赖于使用finalize方法回收任何短期的资源，这是因为很难知道这个方法什么时候才能够调用。
如果某个资源需要在使用完毕后立刻被关闭，那么就需要由人工来管理。对象用完时，可以应用一个close方法来完成相应的清理操作。后续会说到。
```

包

```xml
Java允许使用包将类组织起来。借助于包可以方便地组织自己的代码，并将自己的代码与别人提供的代码库分开管理。
使用包的主要原因是确保类名的唯一性。
嵌套的包之间没有任何关系。例如：java.util包与java.util.jar包毫无关系。每一个都拥有独立的集合。
```

类的导入

```xml
第一种方式是在每个类名之前添加完整的包名
java.time.LocalDate today = java.time.LocalDate.now();
第二种是使用import语句：
	import java.util.*
	import java.util.LocalDate
注意：只能使用星号(*)导入一个包，而不能使用import java.* 或 import java.*.*导入以java为前缀的所有的包。
```

静态导入

```xml
import static java.lang.System.*;
就可以使用System类的静态方法和静态域
```

将类放入包中

```xml
package com.a.b.c;//放在源文件的开头
如果没有源文件中放置package语句，这个源文件中的类就被放置在一个默认包(default package)中。默认包是一个没有名字的包。
```

类设计的技巧

```xml
一定要保证数据私有
一定要对数据初始化
不要在类中使用过多的基本类型
不是所有的域都需要独立的域访问器和域更改器
将职责过多的类进行分解
类名和方法名要能够体现它们的职责
优先使用不可变的类
```
